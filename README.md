# 🌺 Laboratory Work 2-A: Philippine Plant Species Image Classification

## 📋 Project Overview

This project is a comprehensive exercise in building and documenting a machine learning (ML) model from scratch using **Google Teachable Machine**. The goal is to create a functional image classifier capable of identifying **20 different but related plant species** (such as medicinal plants, herbs, ornamental flowers, and flowering plants).

<br>

## 🎯 Purpose of the Image Classification Model

The primary purpose of this image classification model is to **automate the identification and differentiation of 20 related plant species** using visual data. 

By analyzing a dataset of over **5,000 images**, the model learns to recognize specific botanical features such as:
- Leaf shapes
- Floral structures
- Various lighting conditions

The model provides **accurate, confidence-based predictions** that can be validated through performance metrics like confusion matrices and accuracy scores. 

Ultimately, the project produces an **exported, functional machine learning model** that is documented and ready for integration into digital applications.

QUESTION : Written explanation of why you chose those values? 
  -Epochs (50)
    - An epoch is one full pass of the entire training dataset through the neural network.
    Why 50? 50 is the default "sweet spot." It is high enough to allow the model to learn the distinct features of the 20 plant species (like leaf shape or petal color) without being so high that the model              "overfits" (memorizes the photos instead of learning general patterns).

  -Batch Size (16)
    - This is the number of training examples utilized in one iteration. Instead of looking at all 5,000 images at once, the model looks at them in groups of 16. Why 16? A smaller batch size like 16 generally helps     the model converge (find the right answers) more reliably. It also prevents the browser from crashing due to memory limits when handling a large project with 20 different classes.

Model Training Experience - Question Responses :


1. How did the number of images per class affect your model's accuracy?
The number of images per class had a significant impact on the model's overall performance and accuracy. With over 5,000 images distributed across 20 plant species, the model achieved a 100% accuracy rate, indicating that each class had sufficient training examples. Having more images per class allowed the model to learn various angles, lighting conditions, and background variations of each plant species. This robust dataset size helped the model generalize well and avoid overfitting to specific image characteristics.
<br>

3. Which plant species were most commonly misclassified and why?
Based on the confusion matrix, all plant species achieved 100% accuracy with no misclassifications occurring during testing. This exceptional performance suggests that the 20 plant species had sufficiently distinct visual features that the model could easily differentiate. However, in typical scenarios, species with similar colors or petal structures (such as Philippine Violet and Sky Flower Plant, both having purple flowers) would be most prone to confusion. The high-quality dataset and adequate training likely prevented these potential misclassifications from occurring.
<br>

5. How did changing the epochs, batch size, or learning rate affect the training results?
The choice of 50 epochs provided sufficient training time for the model to learn distinctive features without overfitting to the training data. Using a batch size of 16 allowed for stable convergence while managing browser memory constraints effectively with 20 different classes. The default learning rate proved optimal as evidenced by the low loss values and 100% accuracy achieved. These parameter choices created a balance between training efficiency and model performance, resulting in excellent classification results.
<br>
6. What challenges did you encounter during dataset collection and labeling?
Collecting over 5,000 images for 20 different Philippine plant species required significant time and effort to ensure quality and variety. Ensuring proper labeling accuracy was crucial, as mislabeled images during training would have taught the model incorrect associations. Finding images with diverse backgrounds, lighting conditions, and angles for each species was challenging but necessary for model robustness. Additionally, some rare species like Waling-Waling and Rafflesia had limited available images, requiring careful curation from various sources.
<br>
7. If you were to improve your model, what specific changes would you make and why?
I would increase the diversity of the dataset by including images of plants at different growth stages and in various environmental conditions. Implementing data augmentation techniques such as rotation, flipping, and brightness adjustments could further improve the model's robustness. Testing the model with more real-world scenarios, including partially obscured flowers or multiple species in one image, would better validate its practical applicability. Finally, increasing the number of epochs to 75-100 and experimenting with different learning rates could potentially refine the model's feature detection capabilities even further.
